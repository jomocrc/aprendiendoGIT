Config file location

Action

    --unset               remove a variable: name [value-pattern]

    --get-color           find the color configured: slot [default]

Type
    -t, --type <>         value is given this type

FLUJO DE TRAVAJO EN GIT

1º ordenador (git add)
2º etapa de stage (git commit)
3º commit (git push)
4º servidor


con (git status) veremos el estado en el que sencuentran nuestros archivos
si salen en rojo quiere decir que ese archivo no esta siendo segido por git
en el editor de visual studio nos lo marcara con una (U) a la derecha del nombre del archivo
Para poder comenzar a segir un archivo devemos añadirlo con el comando
(git add nombre del archivo quequeremos seguir) despues de esto ejecutamos de nuevo (git status)
y podremos ver como nos indica en verde (newfile: nombre del archivo) al estar en verde nos indica
que este archivo esta listo para se comprometido o comiteado.


para comprometer nuestros archivos y pasarlos a la etapa de (commit) devemos de ejecutar 
(git commit -m "mensaje") el mensaje deve de tener sentido con los cambios realizados para posteriormente
poder comprender al instante que hemos echo dentro de este commit

Eliminando archivos
(rm archivo) rm segido del nombre del archivo que queremos eliminar
si ejecutamos (git status) podremos comprovar que el archivo que emos eliminado nos aparece en rojo
de esta forma  (deleted: archivo2.txt). despues devemos añadir a la etapa de stage los cambios par que
se elimine
(git add archivo2.txt)
tambien lo podemos hacer directamente para ahorrarnos un paso con el comando
(git rm archivo2.txt)
finalmente deveriamos comitearlo

Restaurando un archivo
podremos recuperar el archivo antes de aver echo un commit despues no se puede
para restaura un archivo eliminidado con el comando
git restore archivo2.txt
